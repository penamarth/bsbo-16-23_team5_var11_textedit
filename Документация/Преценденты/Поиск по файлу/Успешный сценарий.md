### Успешнй сценарий: Поиск по тексту

1. Пользователь вводит запрос "data" в интерфейсе редактора, указав:
   - caseSensitive = false
   - wholeWord = true
   - regex = false

2. Пользователь запускает поиск, и Editor вызывает метод:
   searchText("data", caseSensitive=false, wholeWord=true, regex=false).

3. Editor получает текст документа через Document.getText(), который возвращает, например:
   "The data is stored in the database...".

4. Editor передаёт строку и параметры поиска в SearchAnaliz.searchText(fullText, caseSensitive, wholeWord, regex).

5. SearchAnaliz выполняет поиск:
   - Приводит текст и запрос к одному регистру (caseSensitive = false).
   - Ищет только целые слова (wholeWord = true).
   - Не использует регулярные выражения (regex = false).
   - Находит совпадение "data" на позиции 4–8.
   - Игнорирует "data" внутри "database", так как это не отдельное слово.

6. SearchAnaliz формирует список совпадений, например:
   [(start=4, end=8, text="data")]
   и возвращает его в Editor.

7. Editor передаёт документ и список совпадений в ViewText.draw(document, matches) для отображения подсветки.

8. ViewText отрисовывает документ с выделением найденных совпадений.

9. Пользователь видит текст с подсветкой и список совпадений.
